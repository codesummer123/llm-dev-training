## 3. 踩过的坑

### 3.1 把 `build_model()` 和 `create_agent()` 的职责混在一起

最初对 `create_agent` 的理解偏向“帮忙创建模型和读配置”，但实际上：

- 模型初始化是 `build_model()` 的职责
- `create_agent()` 更关键的价值在于组织 agent loop、工具调度、消息回填和最终 orchestration

这是 Day1 最重要的概念修正之一。

### 3.2 注释和真实代码行为一度不一致

一开始 `run_demo()` 的注释写的是“提炼最终回答”，但代码返回的是原始响应结构。  
后续已经修正为返回最终消息内容。

这个问题虽然小，但很典型：

- 注释如果和代码行为不一致，会误导自己和未来维护者

### 3.3 测试输入与实现语言约定不一致

在把难度参数改成中文后，测试仍然保留了 `medium`，导致：

- 手动运行代码看起来没问题
- 但自动化测试暴露出行为契约不一致

这次踩坑很有价值，因为它提醒了一个重要工程原则：

`代码约定一旦变化，测试和文档也必须同步更新。`

### 3.4 测试运行方式依赖工作目录

当前测试依赖在 `code/Day1` 目录下运行。  
这在当前训练阶段可以接受，但后面项目复杂起来时，建议逐步减少这种“工作目录耦合”。

4. 很多 Agent 问题，本质上是上下文工程问题
