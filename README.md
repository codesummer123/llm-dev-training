# 训练方法与总规划

整理日期：2026-03-30

这份文档用于固化本轮训练的长期方法论、项目主线、阶段划分和协作方式，避免随着上下文变长而偏离训练目标。

## 1. 训练总目标

本轮训练不以“看过多少文档”为目标，而以“能独立设计、实现、调试、解释一个复杂智能体系统”为目标。

希望最终达到的能力：

- 熟练使用 `LangChain`、`LangGraph`、`Deep Agents`
- 能根据任务特征做框架选型，而不是只会调用 API
- 能设计带状态、可恢复、可审计、可扩展的 Agent 系统
- 能看穿 AI 领域概念包装，抓住底层结构与工程本质

## 2. 核心训练思想

### 2.1 从“知识点驱动”升级到“项目主线驱动”

只按知识点逐章学习，适合入门，但不利于形成系统设计能力。  
本轮训练采用：

- 项目主线驱动
- 官方文档校准
- 每日小步快跑
- 高频反馈闭环

也就是说，我们不是孤立地学 `tools`、`memory`、`graph`，而是围绕一个逐步进化的真实项目，学习这些抽象为什么存在、什么时候该用、怎么组合。

### 2.2 What / Why / How

每天的训练内容都必须覆盖三层：

- `What`：这个概念是什么
- `Why`：为什么需要它，它解决了什么问题，不用它会怎样
- `How`：代码怎么写，调试时看什么，常见坑是什么

### 2.3 第一性原理

每个框架能力都要回到最底层问题：

- 模型到底负责什么
- 工具到底负责什么
- 状态到底负责什么
- 上下文为什么决定 agent 成败
- 图编排为什么比“单次调用”更适合复杂任务
- 子代理为什么本质上是在做上下文隔离和职责分治

### 2.4 类比学习

为了更快吸收新概念，训练中会不断做类比：

- LangChain Agent 类比为“带工具调用的推理循环”
- LangGraph 类比为“可恢复状态机 + 工作流引擎”
- Deep Agents 类比为“复杂任务代理的执行外骨骼”

## 3. 总项目主线

本轮训练的总项目主线是：

`从 0 到 1 构建一个可进化的智能研究与执行系统`

这个系统会分阶段演进：

1. 先用 `LangChain` 做出一个最小可用的工具型 Agent
2. 再用 `LangGraph` 把它升级成带状态、可恢复、可中断的图式系统
3. 最后用 `Deep Agents` 把它升级成支持子代理、文件系统、长期记忆、复杂任务分解的重型 Agent

这条主线的价值是：

- 同一个项目不断进化，知识复用率高
- 每个新框架都不是“另起炉灶”，而是“解决上一阶段的痛点”
- 能清晰理解三者的边界与分工

## 4. 阶段划分

## 4.1 第一阶段：LangChain 基础与 Agent 抽象

目标：

- 吃透模型、消息、工具、提示词、结构化输出、基础 memory
- 能独立写出一个可用的工具型 Agent

重点：

- `create_agent`
- model / message / tool 三元组
- system prompt
- tool schema
- structured output
- short-term memory
- streaming
- middleware 与 context engineering 入门

## 4.2 第二阶段：LangGraph 编排与运行时

目标：

- 从“能跑”升级到“可恢复、可中断、可编排”

重点：

- state / node / edge / reducer
- Graph API / Functional API
- persistence
- durable execution
- interrupt / resume / replay / fork
- subgraph

## 4.3 第三阶段：Deep Agents 复杂任务系统

目标：

- 掌握复杂任务拆解、上下文隔离、文件系统、长期记忆、subagent 协作

重点：

- `create_deep_agent`
- backend
- subagents
- skills
- sandboxes
- long-term memory
- CLI / MCP / frontend 认知

## 4.4 第四阶段：工程化与外部中间件

目标：

- 引入数据库、缓存、持久化、容器化与更接近生产环境的训练

可引入组件：

- `PostgreSQL`
- `Redis`
- 向量库
- Docker 容器
- LangSmith tracing

原则：

- 先掌握核心抽象，再引入中间件
- 中间件服务于理解，不服务于堆技术名词

## 5. 每天 2 小时的推荐结构

- 15 分钟：复盘前一天，口述或书写 What / Why / How
- 25 分钟：精读当天官网关键内容
- 55 分钟：自己写代码
- 15 分钟：调试、自测、补日志
- 10 分钟：回答问题与输出总结

注意：

- 主时间必须放在“亲手写”和“解释为什么这样写”
- 单日任务必须能收口，避免只学概念不产生产物

## 6. 每日产物规范

每天固定产出三类内容：

## 6.1 `doc`

文件名规范：

- `DayN-训练内容.md`

固定板块：

- 学习目标
- What
- Why
- How
- 代码设计说明
- 验收标准
- 常见坑
- 类比理解

## 6.2 `code`

目录规范：

- `code/DayN/`

建议结构：

```text
code/DayN/
  README.md
  main.py
  prompts.py
  tools.py
  tests/
    test_smoke.py
```

规则：

- 我负责搭脚手架
- 你负责完成核心实现
- 我负责审查代码、指出缺陷、给出优化建议

## 6.3 `answer`

文件名规范：

- `DayN-问题.md`

固定板块：

- 常见问题
- 面试问题
- 开脑问题

其中“开脑问题”用于训练抽象能力、边界判断和系统设计能力。

## 7. 协作方式

训练按天推进，不一次性生成全部内容。

每日节奏：

1. 我生成当天的 `doc / code / answer`
2. 你完成代码实现与思考题
3. 我进行审查，重点看：
   - 逻辑是否正确
   - 抽象是否合理
   - 是否符合官方推荐思路
   - 是否有隐藏 bug / 边界问题 / 可维护性问题
4. 我给出修改建议
5. 只有在当天内容基本过关后，才进入下一天

## 8. 训练中的判断标准

每个知识点按四个层次训练：

- `L1 认知`：能解释概念、边界、适用场景
- `L2 使用`：能按官方示例独立跑通
- `L3 改造`：能基于示例做定制化改造
- `L4 设计`：能针对真实任务做系统设计与选型

我们的目标不是停在 L2，而是稳定推进到 L3 / L4。

## 9. 这轮训练真正要建立的能力

除了掌握 LangChain 系列框架，更重要的是建立这些底层能力：

- 抓本质的能力
- 把复杂问题拆成可执行步骤的能力
- 面向失败与恢复去设计系统的能力
- 把不确定问题转化为可验证工程对象的能力
- 在概念嘈杂环境中保有清晰判断的能力

一句话概括：

`我们练的不是“怎么调用库”，而是“如何把复杂问题做成可执行、可恢复、可验证的智能系统”。`

## 10. 当前执行顺序

接下来按这个顺序推进：

1. 固化这份总规划
2. 从 Day1 开始，建立第一性原理与最小 Agent
3. 逐天积累，逐阶段升级
4. 每天完成后做审查与复盘，再决定是否进入下一天
